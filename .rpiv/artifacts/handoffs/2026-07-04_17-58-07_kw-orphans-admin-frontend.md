## Learnings

- **segment_keywords has no language field.** The 74.6% RU skew is implicit — all demand/stop/synonym phrases for RU-local segments are in Russian. EN phrases exist for international segments (Bali, Vietnam, Thailand, etc.). This matters for any future i18n work on keywords.
- **Matcher (_tag_new_channels) is name-only.** It reads `chat_username` and `title`, checks against the `cities` table, and sets `auto_matched_city_id`. It never inserts new cities. Unknown city names are silently dropped. There is no URL/content-based geo detection.
- **9 of 830 orphans (1.1%) were RU→EN transliteration gaps.** The matcher's exact substring pass would have caught these if the channel name were in Russian. The fuzzy pass (0.95/0.85 thresholds) is too strict for transliterated names — "samui" vs "Самуи" has edit distance way above any reasonable threshold.
- **Fuzzy false positives exist** — short tokens with "nn" match Nizhny Novgorod (@nnw_chat "Neural Chat", @NNR_chat, @nnmidletschat). Threshold 0.95 for <5 chars is the culprit. Known tech debt, not addressed in this session.
- **Admin static is committed to repo.** Vite output in `app/admin/static/` is tracked by git. The old JS/CSS files (index-mfw3riJB.js, previous CSS) should be git-removed after this commit to avoid stale asset accumulation.
