# Phase Review Skill

Автоматический конвейер качества после завершения фазы разработки.

## Процесс

### Шаг 1: Тесты
```bash
pytest tests/ -v
```
- ❌ Тесты не прошли → остановка. Показать какие упали. Предложить исправление.
- ✅ Тесты прошли → Шаг 2.

### Шаг 2: Code Review
Запустить `artifact-code-reviewer` на диффе текущей фазы:
```
Проверь дифф между последним тегом (phase-N-1-done) и HEAD. 
Применяй правила из CODING_STYLE.md:
- Функции ≤ 30 строк
- Ранний возврат вместо вложенных if
- Явные имена переменных
- Комментарии: ПОЧЕМУ, не ЧТО
- Без except Exception
- Без Any
- Все публичные функции с type hints

Верни: blocker / concern / suggestion
```

### Шаг 3: Авто-исправление
- **suggestion** — исправить автоматически (нейминг, форматирование)
- **concern** — исправить если тривиально, иначе показать владельцу
- **blocker** — показать владельцу, запросить решение

### Шаг 4: Повторные тесты
```bash
pytest tests/ -v
```

### Шаг 5: Завершение
- ✅ Все тесты зелёные
- ✅ Blocker'ов нет
- → `git add -A && git commit -m "phase-N: описание" && git tag phase-N-done && git push --tags`

## Правила
- Не менять бизнес-логику без подтверждения
- Blocker = стоп, ждать решения владельца
- CODING_STYLE.md — единственный источник правил
- После исправлений всегда перезапускать тесты
