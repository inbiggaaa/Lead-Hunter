## Контекст

Анализ ядра (12.07.2026, живые данные прода на `main`) показал: архитектура конвейера здоровая (recall-first правила → reality-фильтр → blocking-LLM → sender с дедупом/лимитами/DLQ), но продуктовый результат плохой — **precision 17%** по фидбеку (44 👍 / 215 👎). Причины сконцентрированы, а не размазаны:

1. **«Жирная голова» FP:** `moto-purchase` (370 Pass 1, precision 0%), `car-purchase` (198), `moto-sale` (321) — ~60% входного объёма при ~нулевой полезности. Корень: «продам/продаю» возвращены как demand-фразы (коммит `1abf5fe`), а buy/supply-сегменты минуют Pass 3 (B4). «Кто продает двушку?» матчится в moto-purchase + car-purchase. Часть «восстановленных FN» в eval-отчёте — на деле эти же ложняки.
2. **Петля фидбека разомкнута:** 👎 копятся (259 оценок от 3 пользователей), но никуда не влияют — ни стоп-слова, ни промпт, ни карантин сегментов (massage 0/8, design 0/16, fitness 0/7, repair 1/9).
3. **Recall не измеряется вообще:** весь тюнинг смотрит на precision; backlog `stats:unmatched` упёрся в кап 10000; шаблон ручной разметки из C1 так и не заполнен.
4. **Blocking-LLM — единственный precision-барьер без мониторинга:** fail-open никак не считается; деградация DeepSeek молча превращает систему в «правила без судьи». Pass-rate ~42% (388 DEMAND из 918 за 2 дня). Репосты одного объявления в N чатов = N одинаковых LLM-запросов (13,4 млн токенов/нед).
5. **Reality-фильтр не откалиброван:** в `delivery` блокирует 32/32 (100%), в `car-purchase` 98/147 — не отличить «спасает» от «душит». Обратно: сегменты без domain-слов проходят насквозь с тихим `logger.debug`.
6. **Precision-метрика испорчена старыми slug'ами:** ~20 строк отчёта (crypto 14/54, logistics 1/25, education 0/18…) — фидбек эпохи каталога 29 сегментов, тянет цифру вниз и ничего не говорит о текущих правилах.
7. **Латентность доставки не измеряется:** Hot 5–10 мин, Warm 50 мин, Cold 2,5 ч — лид «срочно, сегодня» из Cold приходит мёртвым. Решение #80 (поллинг без вступления) принято без данных о цене латентности.

**Цель плана:** precision ≥ 50% по фидбеку при неухудшении recall (проверяется разметкой B3), измеримая латентность, замкнутая петля качества. Архитектурные перестройки (скоринг, event-push, дистилляция) — только по итогам измерений, gate-условия прописаны в Фазе C.
