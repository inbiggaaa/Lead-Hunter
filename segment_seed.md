## 0. ЛОГИКА ОБРАБОТКИ СООБЩЕНИЯ (применять к каждому сообщению)

**Шаг 1.** Есть хотя бы одна ПОЗИТИВНАЯ фраза спроса из категории?
→ НЕТ: игнорировать сообщение.
→ ДА: сообщение становится кандидатом.

**Шаг 2.** Прогнать кандидата через МИНУС-СЛОВА (раздел 1) и СТРУКТУРНЫЕ СИГНАЛЫ (раздел 2).

**Шаг 3. Правило приоритета:**
- Если сильный глагол спроса стоит В НАЧАЛЕ фразы (ищу / нужен / нужна / нужны / сниму / возьму / посоветуйте / подскажите / хочу / где / кто делает) — он ПЕРЕБИВАЕТ слабые маркеры оффера. Оставляем.
- В остальных случаях наличие маркера оффера ГАСИТ сообщение.
- Вопросительный знак усиливает «спрос».
- Цена + контакт/ссылка/@username + хештеги усиливают «оффер».

**Шаг 4.** Короткие anchors — фразы с высокой вероятностью ложного срабатывания. Матчатся ТОЛЬКО при наличии дополнительного контекстного сигнала спроса (глагол спроса в начале, вопросительный знак, «подскажите»/«посоветуйте»).
