При N=12 терялись подтверждённые лиды с разлётом слов 15-17 токенов:
- «Всем привет ищем приватный тур … как туда добраться помимо такси?» (👍, taxi-transfer, kw «ищу такси», разлёт 17)
- «…Скажите сколько брать денег … Хочу валюту поменять…» (👍, currency-exchange, kw «поменять деньги», разлёт 15)

а также живые заявки из decisions: «кто сдает автомобиль в аренду?» (car-rental),
«где вменяемо сейчас проще найти нормальный курс?» (currency-exchange), «на экскурсию
Куда можно съездить!» (excursions).

Гистограмма минимального окна для 113 потерянных при N=12: ≤16 → 25, ≤20 → 5,
≤24 → 8, ≤32 → 12, ≤48 → 3, разлёт за пределами 300 символов → 60.

## Асимметрия цены ошибки

Rules стоят ДО blocking-LLM: потерянный на этом этапе лид невосполним, а
пропущенный оффер уходит в LLM, который его режет (цена — токены). Возвращаемые
окном 20 тексты — почти целиком спам, который LLM уже отклонял (рассылка «нужен
персонал на Binance» ×10, реклама Сбера, courier-обменник) — user-facing FP они
не создают. Потерянные же при N=12 — реальные заявки.

## Что режет окно 20 (80 матчей)

Все 60 «дальних» разлётов — длинные офферные посты, где слова demand-фразы
случайно встречаются в разных абзацах (прайсы, реклама туров, реф-спам), плюс
20 средних разлётов того же характера. gained=0 подтверждает: окно только
ужесточает, mixed-form матчинг не введён (три формо-альтернативы сохранены).
