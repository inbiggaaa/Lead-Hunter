# A2 — офлайн-diff классификатора: обход stop-слов через «?» закрыт

**Дата:** 09.07.2026
**Ветка:** `audit/fable-fixes`
**Корпус:** 1000 сообщений из baseline 0.1 (`docs/eval/baseline_2026-07.decisions.jsonl` + `baseline_2026-07.unmatched.jsonl`, по 500 из каждого).
**Методика:** каждый текст прогнан старым классификатором (снимок `git show 439c3fe:app/userbot/classifier.py`) и новым; keywords/stops — прод-набор из БД (read-only). Скрипт и сырые результаты — scratchpad `a2_diff.py` / `a2_diff_raw.jsonl`.

## Суть фикса

Старый `_has_demand_signal` возвращал True при **любом «?»** в тексте, а Pass 2
применял stop-слова только `if not has_demand_context` — любая реклама с
вопросом-хуком («Ищешь байк в аренду?») обходила все стопы. Тот же флаг включал
pre-tag boost в poller — голый вопрос в предтегированном канале становился лидом.

Новое разделение:

| Уровень | Что входит | Где используется |
|---|---|---|
| `_has_strong_demand_signal` | паттерны DEMAND_SIGNAL_PATTERNS + short-anchor-контекст, **без голого «?»** | Pass 2 (переопределение stop-слов), pre-tag boost в `_poll_channel` |
| `has_demand_context` (weak) | strong ИЛИ «?» | Pass 3 (структурные сигналы) — поведение не менялось |

## Вывод

Фикс убирает **только офферы** (20 полных + 4 частичных из 520 старых матчей,
~4% корпуса — все подтверждённо реклама), не теряет ни одного лида и не
добавляет новых матчей. Строгие demand-паттерны («подскажите», «где снять…?»
и т.п.) продолжают переопределять stop-слова — покрыто тестами
`tests/test_stop_bypass.py` (7 шт.).
