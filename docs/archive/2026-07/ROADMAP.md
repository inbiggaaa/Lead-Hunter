# ROADMAP — Анализ проблем (01.07.2026)

## 🔴 Классификация

### 1. Лемматизация ломает матчинг
**Проблема:** pymorphy3 выбирает не ту часть речи. «Наличные» → «наличный» (прил.) вместо «наличные» (сущ.) → матчит «Наличный обмен» (реклама).
**Идея:** принудительный POS-тег для многозначных слов в keywords.

### 2. Нет learning-цикла
**Проблема:** каждый false positive чиним вручную в коде.
**Идея:** кнопка «👎 Не релевантно» в уведомлении → авто-добавление стоп-слова / минус-фразы.

### 3. Offer-детектор всё ещё дырявый
**Проблема:** регулярками не покрыть все форматы рекламы. EXASIA — нет цены/телефона, только «Добро пожаловать». Kymco — цена без слова «цена».
**Идея:** эвристики по структуре (маркированные списки, цены VND/донг, «Blue Card»).

## 🔴 Discovery

### 4. 38-дневный цикл — очень долго
**Проблема:** при 90-сек паузе канал может ждать открытия месяц.
**Идея:** приоритизация Tier 1 стран → Tier 2 → Tier 3. Не ждать полный цикл.

### 5. Качество поиска
**Проблема:** шаблон `{город} {слово}`. «Барахолка Нячанга», «Общага Дананг» — обратный порядок не ловится.
**Идея:** поиск по городу БЕЗ community-слова → фильтрация результатов по заголовку.

### 6. 1165 каналов без привязки к городу
**Файл:** `no_city_channels.csv` (в корне проекта)
Привязаны только к стране. При mode='cities' — доставляются всё равно (city_id=NULL → фильтр пропускает). Это не баг, но шум.

### 6a. Авто-теггинг городов не находит все каналы
**Проблема:** `_tag_new_channels()` ищет точное совпадение `city_name_ru in title_lower`. Но каналы пишут города с опечатками («Анталия» vs «Анталья»), транслитом («Alanya»), в падежах («Барахолка Нячанга» вместо «Нячанг»), с ошибками в slug.
**Идея:** fuzzy matching, поиск по name_en тоже, нормализация окончаний.

## 🟡 Архитектурные заделы (Фаза 1+)

### 15. Сон аккаунтов по ночи региона (после 3-го аккаунта)
При появлении 3-го аккаунта пересмотреть `_get_sleep_start_hour`: вместо равномерного
разноса — привязать сон каждого аккаунта к ночи его аудитории (Вьетнам UTC+7 / Москва
UTC+3), чтобы отдых падал на мёртвые часы региона. ТРЕБУЕТ предварительно: гео-раздачу
каналов в `_distribute` по региону (сейчас round-robin — привязка аккаунт↔регион не
работает). Не трогать `_distribute` ради переброски (риск #3) — речь только о том, какие
каналы какому аккаунту, не о переносе нагрузки со спящего на активного.

## 🚀 Операционные планы (после стабилизации)

### Повысить частоту опросов (через несколько дней)
Сейчас Hot=10мин (600с). Владелец хочет быстрее уведомления.
Снизить `hot_interval_base` 600→300 (5мин) или 180 (3мин).
**УСЛОВИЯ перед снижением:**
- Post_ban истёк (48ч после последнего FloodWait, ≈04.07 ~14:06 MSK)
- Появился 3-й аккаунт (распределение нагрузки на троих)
- Сутки без FloodWait на текущих настройках
**ПОРЯДОК:** сначала 3-й аккаунт → потом снижение интервала.

### Третий аккаунт Telegram (через 4-5 дней, ждём SIM)
Карантин 1-2 недели, нагрузка 5-10% от бюджета.
`_get_effective_interval` (7мин при 3+) и `_get_sleep_start_hour` (разнос сна)
уже готовы к 3 аккаунтам — реализованы в Задаче 1.3 и 1.2.
Снимает последний системный риск (одна точка отказа, аудит 8.1).
После стабилизации 3-го → снижать Hot-интервал (см. выше).
