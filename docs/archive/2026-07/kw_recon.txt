      id            BIGINT PK

      {segment_slug: {"demand": [...], "stop": [...], "synonym": [...]}}
    + universal_stops (segment_id=NULL, keyword_type='stop')

(4) ФУНКЦИЯ РЕЛЕВАНТНОСТИ
============================================================================

    app/userbot/classifier.py:202 — classify_message(text, segment_keywords, universal_stops)

    3 прохода:
      Pass 1 — demand + synonym: word-boundary match (regex), лемматизация
               (pymorphy3), fuzzy fallback для фраз из 4+ слов (≥70% слов).
      Pass 2 — stop: универсальные + специфичные сегменту. Гасит совпадение,
               ЕСЛИ нет сильного demand-сигнала.
      Pass 3 — структурные сигналы: глагол спроса В НАЧАЛЕ (ищу/нужен/...) →
               перебивает оффер; цена+контакт → оффер, гасит без сильного demand.

    Точка вызова: poller.py:368 — для каждого сообщения.
    Кэширование: Redis class:cache:{message_hash} (TTL 60с).
