## 1. МЕХАНИЗМ привязки

### Где присваивается auto_matched_city_id

**`app/userbot/poller.py:1169` — `_tag_new_channels()`**

Входы: `chat_username` + `title` (склеенные, lowercased). URL/ссылка/описание — **НЕ используются**.

Логика:
- Pass 1: точное вхождение названия города (name_ru и name_en) в `chat_username.lower() + " " + title.lower()`
- Pass 2: fuzzy-матч через `SequenceMatcher` (порог 0.85 для слов ≥5 букв, 0.95 для <5)

Обновление: только каналы с `auto_matched_country_id IS NOT NULL AND auto_matched_city_id IS NULL`.

### Одно-сити vs мульти-сити

`auto_matched_city_id` — скаляр (одно значение). Если найдено >1 города:
- `auto_matched_city_id` = первый найденный (line 1258)
- все найденные → вставляются в `channel_cities` (line 1269-1274)
- старые строки `channel_cities` удаляются (line 1266)

### Справочник городов

**91 страна, 227 городов.** Алиасы: ищет по `name_ru` И `name_en` для каждого города. Если name_en совпадает с name_ru — дубль не добавляется. Алиасы жёстко привязаны к названиям из БД — нет отдельной таблицы синонимов/транслитераций. Fuzzy (Pass 2) частично компенсирует это (SequenceMatcher).
