## Контрольные точки

### Фаза 0 — Подготовка сервера

- [ ] SSH работает по ключу (не по паролю)
- [ ] `docker --version` и `python3 --version` отвечают
- [ ] Swap 2GB настроен
- [ ] Токены получены: BOT_TOKEN, USERBOT_API_ID, USERBOT_API_HASH
- [ ] Контейнеры стартуют с memory limits
- [ ] UFW: только порты 22 (и 80/443 если веб)

### Фаза 1 — Скелет проекта

- [ ] `/start` в Telegram → бот отвечает
- [ ] Контейнеры живут после `docker compose up`
- [ ] i18n работает (ru/en) — тексты из `locales/`
- [ ] Кнопки выбора языка на приветственном экране
- [ ] Onboarding wizard запускается при первом `/start`

### Фаза 2 — БД и модели

- [ ] После `/start` в таблице `users` появляется запись
- [ ] Все 15 таблиц созданы миграцией Alembic
- [ ] Миграция обратима (downgrade работает)
- [ ] Seed-данные загружены (segment_seed.md → seed_catalog.py)
- [ ] Unit-тесты на модели и keyword_matches проходят
- [ ] Индексы созданы (особенно idx_user_sub_lookup)

### Фаза 3 — Inline-кнопки

- [ ] Главное меню из 9 кнопок работает
- [ ] Добавление/удаление слов — лимиты Free срабатывают
- [ ] Добавление/удаление каналов — лимиты Free срабатывают
- [ ] Приватный канал → сообщение + алерт владельцу
- [ ] FSM-воронка: направление → страна → география → города → подтверждение
- [ ] Trial активируется при первом прохождении воронки
- [ ] Экран триала → «Показать 10 последних запросов»
- [ ] Экран ⚙️ Настройки: дайджест уведомлений, игнор-лист, CSV-экспорт (Business)

### Фаза 4 — Userbot + Classifier

- [ ] Userbot авторизован (session-файл)
- [ ] Poller слушает каналы без вступления (GetHistory, умный по tier'ам)
- [ ] Импорт спаршенных списков каналов через админку
- [ ] `classifier.classify_message()` отрабатывает на каждом сообщении
- [ ] Трёхпроходный алгоритм работает (demand → stop → structural)
- [ ] Личные keywords матчатся независимо
- [ ] Срочные (🔥) детектятся
- [ ] Результат классификации попадает в лог

### Фаза 5 — Очередь и рассыльщик

- [ ] Paid-формат: полный, кнопки «Ответить»
- [ ] Throttle 25/сек работает
